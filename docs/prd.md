# 需求文档

## 1. 应用概述

- **应用名称**：Broken of Love 链接导航页
- **应用描述**：一个单页网页，包含个人头像与标题展示，以及小红书、微博、抖音、B站四个平台的链接群，每个平台链接群支持收拢与展开操作。

---

## 2. 页面结构与核心功能

### 2.1 页面总览

```
单页网页
├── 模块一：个人信息展示区
├── 模块二：小红书链接群
├── 模块三：微博链接群
├── 模块四：抖音链接群
└── 模块五：B站链接群
```

### 2.2 模块一：个人信息展示区

- 展示一个圆形头像图片，使用占位图代替（待用户后续替换）
- 头像下方展示标题文字：Broken of Love
- 布局居中对齐

### 2.3 模块二：小红书链接群

- 模块标题：小红书
- 支持点击标题区域进行收拢 / 展开切换，默认展开
- 展开状态下，展示若干条链接列表，每条链接包含：
  - 链接标题（描述文字）
  - 可点击跳转的链接地址
- 链接内容使用占位示例数据，待用户后续替换

### 2.4 模块三：微博链接群

## 3. 业务规则与逻辑

- 每个平台链接群默认为展开状态
- 点击模块标题区域可切换收拢 / 展开状态，收拢时隐藏链接列表，仅显示模块标题
- 收拢 / 展开状态切换时，应有明显的视觉指示（如箭头图标方向变化）
- 链接点击后在新标签页中打开目标地址

---

## 4. 异常与边界情况

| 场景 | 处理方式 |
|------|----------|
| 头像图片加载失败 | 显示默认占位图或灰色圆形背景 |
| 链接地址为空 | 该条链接不可点击，视觉上置灰处理 |
| 链接群内无链接条目 | 展开后显示「暂无内容」提示文字 |

---

## 5. 验收标准

- 页面包含五个模块，顺序与需求一致
- 模块一正确展示占位头像与标题「Broken of Love」
- 四个平台链接群均可正常收拢与展开，状态切换有视觉反馈
- 链接点击后在新标签页中打开
- 页面整体布局在桌面端正常显示
